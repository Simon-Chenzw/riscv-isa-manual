## 摘要

本文档描述 RV32I 基础整数指令集。RV32I 被设计为足以形成编译器目标并支持现代操作系统环境。该 ISA 还被设计为减少最小实现所需的硬件。RV32I 包含 40 个唯一指令，为 32 位地址空间提供完整的计算能力。

### 涉及的指令格式

#### 基础指令格式 (32位)

**R-type (寄存器-寄存器):**
```
[31:25] funct7 | [24:20] rs2 | [19:15] rs1 | [14:12] funct3 | [11:7] rd | [6:0] opcode
   7位         |    5位      |    5位      |    3位        |   5位   |   7位

示例: ADD x1, x2, x3
funct7=0000000 | rs2=x3 | rs1=x2 | funct3=000 | rd=x1 | opcode=0110011
```

**I-type (立即数):**
```
[31:20] imm[11:0] | [19:15] rs1 | [14:12] funct3 | [11:7] rd | [6:0] opcode
    12位         |    5位      |    3位        |   5位   |   7位

示例: ADDI x1, x2, 100
imm=000001100100 | rs1=x2 | funct3=000 | rd=x1 | opcode=0010011
```

**S-type (存储):**
```
[31:25] imm[11:5] | [24:20] rs2 | [19:15] rs1 | [14:12] funct3 | [11:7] imm[4:0] | [6:0] opcode
    7位          |    5位      |    5位      |    3位        |    5位         |   7位

示例: SW x3, 8(x2)
imm[11:5]=0000000 | rs2=x3 | rs1=x2 | funct3=010 | imm[4:0]=01000 | opcode=0100011
```

**B-type (分支):**
```
[31] imm[12] | [30:25] imm[10:5] | [24:20] rs2 | [19:15] rs1 | [14:12] funct3 | [11:8] imm[4:1] | [7] imm[11] | [6:0] opcode
```

**U-type (上立即数):**
```
[31:12] imm[31:12] | [11:7] rd | [6:0] opcode
    20位          |   5位   |   7位

示例: LUI x1, 0x12345
imm[31:12]=12345000000000000000 | rd=x1 | opcode=0110111
```

**J-type (跳转):**
```
[31] imm[20] | [30:21] imm[10:1] | [20] imm[11] | [19:12] imm[19:12] | [11:7] rd | [6:0] opcode
```

### 主要指令列表

| 指令类型 | 指令 | 格式 | 功能描述 |
|---------|------|------|----------|
| 算术 | ADD | R-type | rd = rs1 + rs2 |
| 算术 | ADDI | I-type | rd = rs1 + imm |
| 逻辑 | AND | R-type | rd = rs1 & rs2 |
| 逻辑 | ANDI | I-type | rd = rs1 & imm |
| 移位 | SLL | R-type | rd = rs1 << rs2[4:0] |
| 移位 | SLLI | I-type | rd = rs1 << imm[4:0] |
| 比较 | SLT | R-type | rd = (rs1 < rs2) ? 1 : 0 |
| 比较 | SLTI | I-type | rd = (rs1 < imm) ? 1 : 0 |
| 加载 | LW | I-type | rd = M[rs1 + imm][31:0] |
| 加载 | LH | I-type | rd = SignExt(M[rs1 + imm][15:0]) |
| 加载 | LB | I-type | rd = SignExt(M[rs1 + imm][7:0]) |

| 分支 | BNE | B-type | if (rs1 != rs2) PC += imm |

| 系统 | EBREAK | I-type | 断点 |

## 程序员模型

RV32I 的程序员可见状态包括32个通用寄存器和程序计数器。对于 RV32I，32 个 `x` 寄存器每个都是 32 位宽，即 `XLEN=32`。寄存器 `x0` 硬连线为全零。通用寄存器 `x1-x31` 保存各种指令解释为布尔值集合、二进制补码有符号二进制整数或无符号二进制整数的值。

还有一个额外的非特权寄存器：程序计数器 `pc` 保存当前指令的地址。

### RISC-V 基础非特权整数寄存器状态

| 寄存器 | ABI名称 | 描述 | 保存者 |
|--------|---------|------|--------|
| x0 | zero | 硬连线零 | - |
| x1 | ra | 返回地址 | 调用者 |
| x2 | sp | 栈指针 | 被调用者 |
| x3 | gp | 全局指针 | - |
| x4 | tp | 线程指针 | - |
| x5-x7 | t0-t2 | 临时寄存器 | 调用者 |
| x8 | s0/fp | 保存寄存器/帧指针 | 被调用者 |
| x9 | s1 | 保存寄存器 | 被调用者 |
| x10-x11 | a0-a1 | 函数参数/返回值 | 调用者 |
| x12-x17 | a2-a7 | 函数参数 | 调用者 |
| x18-x27 | s2-s11 | 保存寄存器 | 被调用者 |
| x28-x31 | t3-t6 | 临时寄存器 | 调用者 |
| pc | - | 程序计数器 | - |

> **注释：**
> 基础整数 ISA 中没有专用的栈指针或子程序返回地址链接寄存器；指令编码允许任何 `x` 寄存器用于这些目的。然而，标准软件调用约定使用寄存器 `x1` 来保存调用的返回地址，寄存器 `x5` 可作为备用链接寄存器。标准调用约定使用寄存器 `x2` 作为栈指针。

## 基础指令格式

在基础 RV32I ISA 中，有四种核心指令格式（R/I/S/U）。所有指令都是固定的 32 位长度。基础 ISA 具有 `IALIGN=32`，意味着指令必须在内存中的四字节边界上对齐。如果目标地址不是 `IALIGN位` 对齐的，则在采用分支或无条件跳转时生成指令地址错位异常。此异常在分支或跳转指令上报告，而不是在目标指令上。对于未采用的条件分支不会生成指令地址错位异常。

> **注释：**
> 当添加具有 16 位长度或 16 位长度的其他奇数倍的指令扩展时，基础 ISA 指令的对齐约束被放宽到两字节边界（即，IALIGN=16）。

RISC-V ISA 在所有格式中将源寄存器（_rs1_ 和 _rs2_）和目标寄存器（_rd_）保持在相同位置以简化解码。除了 CSR 指令中使用的 5 位立即数外，立即数总是符号扩展的，并且通常打包到指令中最左边的可用位，并且被分配以减少硬件复杂性。特别是，所有立即数的符号位总是在指令的第 31 位，以加速符号扩展电路。

### 指令格式详细说明

**R-type 格式（寄存器-寄存器操作）：**
- 用于寄存器之间的算术和逻辑操作
- funct7 和 funct3 字段选择具体操作类型
- 示例：ADD, SUB, AND, OR, XOR, SLL, SRL, SRA, SLT, SLTU

**I-type 格式（立即数操作）：**
- 用于立即数算术、加载指令和JALR
- 12位立即数符号扩展
- 示例：ADDI, SLTI, ANDI, LW, LH, LB, JALR

**S-type 格式（存储操作）：**
- 12位立即数分为两部分以保持寄存器字段位置一致
- 示例：SW, SH, SB

**U-type 格式（上立即数）：**
- 用于构建大常数
- 20位立即数放在高位，低12位为零
- 示例：LUI, AUIPC

## 立即数编码变体

还有基于立即数处理的另外两种指令格式变体（B/J）：

**B-type 格式（分支操作）：**
- 12位立即数编码分支偏移量，以2的倍数计算
- 分支范围：±4 KiB

**J-type 格式（跳转操作）：**
- 20位立即数编码跳转偏移量，以2的倍数计算  
- 跳转范围：±1 MiB

### 立即数类型

每种指令格式产生的立即数：

- **I-immediate**: 12位，符号扩展到XLEN
- **S-immediate**: 12位，符号扩展到XLEN
- **B-immediate**: 13位（隐含最低位0），符号扩展到XLEN
- **U-immediate**: 32位（低12位为0）
- **J-immediate**: 21位（隐含最低位0），符号扩展到XLEN

> **注释：**
> 符号扩展是立即数上最关键的操作之一（特别是对于XLEN>32），在RISC-V中，所有立即数的符号位总是保存在指令的第31位，以允许符号扩展与指令解码并行进行。

## 整数计算指令

大多数整数计算指令对整数寄存器文件中保存的 `XLEN` 位值进行操作。整数计算指令要么编码为使用 I-type 格式的寄存器-立即数操作，要么编码为使用 R-type 格式的寄存器-寄存器操作。对于寄存器-立即数和寄存器-寄存器指令，目标都是寄存器 _rd_。没有整数计算指令引起算术异常。

### 整数寄存器-立即数指令

**ADDI** 将符号扩展的12位立即数加到寄存器 _rs1_。算术溢出被忽略，结果简单地是结果的低XLEN位。`ADDI rd, rs1, 0` 用于实现 `MV rd, rs1` 汇编器伪指令。

**SLTI**（设置小于立即数）如果寄存器 _rs1_ 小于符号扩展立即数（当两者都被视为有符号数时），则在寄存器 _rd_ 中放置值1，否则向 _rd_ 写入0。**SLTIU** 类似，但将值作为无符号数比较。

**ANDI, ORI, XORI** 是逻辑操作，对寄存器 _rs1_ 和符号扩展的12位立即数执行按位AND、OR和XOR，并将结果放在 _rd_ 中。

**移位指令：**
- **SLLI**：逻辑左移（零移入低位）
- **SRLI**：逻辑右移（零移入高位）
- **SRAI**：算术右移（原符号位复制到空出的高位）

移位量编码在I-立即数字段的低5位中。右移类型在第30位编码。

**立即数构建指令：**

**LUI**（加载上立即数）用于构建32位常数，使用U-type格式。LUI将32位U-立即数值放入目标寄存器 _rd_，最低12位填零。

**AUIPC**（将上立即数加到pc）用于构建pc相对地址，使用U-type格式。AUIPC从U-立即数形成32位偏移量，最低12位填零，将此偏移量加到AUIPC指令的地址，然后将结果放在寄存器 _rd_ 中。

### 整数寄存器-寄存器操作

RV32I 定义了几个算术 R-type 操作。所有操作读取 _rs1_ 和 _rs2_ 寄存器作为源操作数，并将结果写入寄存器 _rd_。_funct7_ 和 _funct3_ 字段选择操作类型。

**算术操作：**
- **ADD**：执行 _rs1_ 和 _rs2_ 的加法
- **SUB**：执行 _rs2_ 从 _rs1_ 的减法
- **SLT, SLTU**：分别执行有符号和无符号比较

**逻辑操作：**
- **AND, OR, XOR**：执行按位逻辑操作

**移位操作：**
- **SLL, SRL, SRA**：在寄存器 _rs1_ 的值上执行逻辑左移、逻辑右移和算术右移

### NOP 指令

**NOP** 指令不改变任何架构可见状态，除了推进 `pc` 和增加任何适用的性能计数器。NOP 编码为 `ADDI x0, x0, 0`。

## 控制转移指令

RV32I 提供两种类型的控制转移指令：无条件跳转和条件分支。RV32I 中的控制转移指令**没有**架构可见的延迟槽。

### 无条件跳转

**JAL**（跳转和链接）指令使用J-type格式，其中J-立即数编码以2字节倍数的有符号偏移量。偏移量符号扩展并加到跳转指令的地址以形成跳转目标地址。因此跳转可以目标±1 MiB范围。JAL将跳转后指令的地址（'pc'+4）存储到寄存器 _rd_ 中。

```
JAL x1, label    # x1 = PC + 4, PC = PC + offset_to_label
```

**JALR**（跳转和链接寄存器）间接跳转指令使用I-type编码。目标地址通过将符号扩展的12位I-立即数加到寄存器 _rs1_ 获得，然后将结果的最低有效位设为零。跳转后指令的地址（`pc`+4）写入寄存器 _rd_。

```
JALR x1, x2, 100    # x1 = PC + 4, PC = (x2 + 100) & ~1
```

### 条件分支

所有分支指令使用B-type指令格式。12位B-立即数编码以2字节倍数的有符号偏移量。偏移量符号扩展并加到分支指令的地址以给出目标地址。条件分支范围是±4 KiB。

**分支指令：**
- **BEQ, BNE**：如果寄存器 _rs1_ 和 _rs2_ 相等或不相等则分支
- **BLT, BLTU**：如果 _rs1_ 小于 _rs2_，使用有符号和无符号比较则分支
- **BGE, BGEU**：如果 _rs1_ 大于或等于 _rs2_，使用有符号和无符号比较则分支

```
BEQ x1, x2, label    # if (x1 == x2) PC = PC + offset_to_label
BLT x1, x2, label    # if (x1 < x2) PC = PC + offset_to_label (signed)
BLTU x1, x2, label   # if (x1 < x2) PC = PC + offset_to_label (unsigned)
```

> **注释：**
> 有符号数组边界可以用单个BLTU指令检查，因为任何负索引都会比任何非负边界比较得更大。

## 加载存储指令

RV32I 是加载-存储架构，只有加载和存储指令访问内存，算术指令只操作CPU寄存器。RV32I 提供字节寻址的32位地址空间。

加载和存储指令在寄存器和内存之间传输值。加载编码为I-type格式，存储为S-type。有效地址通过将寄存器 _rs1_ 加到符号扩展的12位偏移量获得。加载将值从内存复制到寄存器 _rd_。存储将寄存器 _rs2_ 中的值复制到内存。

### 加载指令

- **LW**：从内存加载32位值到 _rd_
- **LH**：从内存加载16位值，然后符号扩展到32位后存储在 _rd_ 中
- **LHU**：从内存加载16位值，然后零扩展到32位后存储在 _rd_ 中
- **LB, LBU**：对8位值的类似定义

```
LW x1, 8(x2)     # x1 = M[x2 + 8][31:0]
LH x1, 4(x2)     # x1 = SignExt(M[x2 + 4][15:0])
LB x1, 0(x2)     # x1 = SignExt(M[x2 + 0][7:0])
```

### 存储指令

- **SW, SH, SB**：将寄存器 _rs2_ 的低位32位、16位和8位值存储到内存

```
SW x3, 8(x2)     # M[x2 + 8] = x3[31:0]
SH x3, 4(x2)     # M[x2 + 4] = x3[15:0]
SB x3, 0(x2)     # M[x2 + 0] = x3[7:0]
```

### 内存对齐

无论EEI如何，有效地址自然对齐的加载和存储不应引发地址错位异常。有效地址未自然对齐到引用数据类型的加载和存储（即，有效地址不能被访问的字节大小整除）具有依赖于EEI的行为。

## 内存排序指令

**FENCE** 指令用于按其他RISC-V硬件线程和外部设备或协处理器所观察的那样排序设备I/O和内存访问。可以相对于设备输入（I）、设备输出（O）、内存读取（R）和内存写入（W）的任何组合排序相同的任何组合。

```
FENCE pred, succ
其中 pred/succ 可以是 IORW 的任意组合
```

**FENCE模式：**
- **FENCE**（fm=0000）：普通栅栏
- **FENCE.TSO**（fm=1000）：总存储排序栅栏

FENCE指令的格式：
```
[31:28] fm | [27:24] pred | [23:20] succ | [19:15] rs1 | [14:12] funct3 | [11:7] rd | [6:0] opcode
```

## 环境调用和断点

**SYSTEM** 指令用于访问可能需要特权访问的系统功能，使用I-type指令格式编码。

### 系统指令

**ECALL**：用于向执行环境发出服务请求。EEI将定义如何传递服务请求的参数，但通常这些将在整数寄存器文件的定义位置中。

```
ECALL              # 环境调用
编码: 000000000000 00000 000 00000 1110011
```

**EBREAK**：用于将控制返回到调试环境。

```
EBREAK             # 断点
编码: 000000000001 00000 000 00000 1110011
```

> **注释：**
> ECALL和EBREAK以前名为SCALL和SBREAK。指令具有相同的功能和编码，但被重命名以反映它们可以比调用监管者级操作系统或调试器更一般地使用。

## 提示指令

RV32I 为HINT指令保留大量编码空间，这些指令通常用于向微架构传达性能提示。像NOP指令一样，HINT不改变任何架构可见状态，除了推进 `pc` 和任何适用的性能计数器。实现总是被允许忽略编码的提示。

大多数RV32I HINT被编码为目标寄存器 _rd_=`x0` 的整数计算指令。其他RV32I HINT被编码为具有空前驱或后继集合并且 _fm_=0 的FENCE指令。

### HINT编码示例

```
ADD x0, x1, x2     # HINT: 可能指示x1和x2将被一起使用
ADDI x0, x1, imm   # HINT: 可能指示x1即将被访问
```

> **注释：**
> 这些HINT编码已被选择，使得简单实现可以完全忽略HINT，而是将HINT作为恰好不改变架构状态的常规指令执行。例如，如果目标寄存器是 `x0`，ADD就是HINT；5位_rs1_和_rs2_字段编码HINT的参数。

---

RV32I提供了一个完整、简洁且优雅的基础指令集，能够支持广泛的计算任务。其设计哲学强调简单性、可扩展性和实现效率，为RISC-V架构的更高级扩展奠定了坚实基础。通过40个核心指令，RV32I展示了如何用最少的硬件资源实现完整的32位计算机系统。
