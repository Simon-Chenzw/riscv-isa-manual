## 摘要

本章描述RISC-V标准压缩指令集扩展，名为"C"，通过为常见操作添加短的16位指令编码来减少静态和动态代码大小。C扩展可以添加到任何基础ISA（RV32I、RV32E、RV64I、RV64E）中。通常，程序中50%-60%的RISC-V指令可以用RVC指令替换，从而实现25%-30%的代码大小减少。

### 涉及的指令格式

#### 压缩指令格式 (16位)

压缩扩展使用九种16位指令格式：

**CR格式 (寄存器):**
```
[15:12] funct4 | [11:7] rd/rs1 | [6:2] rs2 | [1:0] op
    4位       |     5位       |   5位    |  2位

示例: C.ADD x1, x2
funct4=1001 | rd/rs1=x1 | rs2=x2 | op=10
```

**CI格式 (立即数):**
```
[15:13] funct3 | [12] imm[5] | [11:7] rd/rs1 | [6:2] imm[4:0] | [1:0] op
    3位       |    1位      |     5位       |     5位        |  2位

示例: C.ADDI x1, 10
funct3=000 | imm[5]=0 | rd/rs1=x1 | imm[4:0]=01010 | op=01
```

**CSS格式 (栈相对存储):**
```
[15:13] funct3 | [12:7] imm | [6:2] rs2 | [1:0] op
    3位       |    6位     |   5位    |  2位

示例: C.SWSP x5, 8
funct3=110 | imm=000010 | rs2=x5 | op=10
```

**CIW格式 (宽立即数):**
```
[15:13] funct3 | [12:5] imm | [4:2] rd' | [1:0] op
    3位       |    8位     |   3位    |  2位

示例: C.ADDI4SPN x8, 16
```

**CL格式 (加载):**
```
[15:13] funct3 | [12:10] imm | [9:7] rs1' | [6:5] imm | [4:2] rd' | [1:0] op
    3位       |     3位     |    3位     |   2位    |   3位    |  2位

示例: C.LW x8, 4(x9)
```

**CS格式 (存储):**
```
[15:13] funct3 | [12:10] imm | [9:7] rs1' | [6:5] imm | [4:2] rs2' | [1:0] op
    3位       |     3位     |    3位     |   2位    |   3位     |  2位

示例: C.SW x10, 8(x11)
```

## 概述

RVC使用简单的压缩方案，在以下情况下为常见的32位RISC-V指令提供更短的16位版本：

* 立即数或地址偏移量很小，或
* 寄存器之一是零寄存器（`x0`）、ABI链接寄存器（`x1`）或ABI栈指针（`x2`），或
* 目标寄存器和第一个源寄存器相同，或
* 使用的寄存器是最流行的8个。

C扩展与所有其他标准指令扩展兼容。C扩展允许16位指令与32位指令自由混合，后者现在能够在任何16位边界上开始，即IALIGN=16。随着C扩展的添加，没有指令可以引发指令地址错位异常。

### 设计目标

RVC在以下约束下设计：每个RVC指令扩展为基础ISA（RV32I/E或RV64I/E）或F和D标准扩展中的单个32位指令。采用此约束有两个主要好处：

* 硬件设计可以在解码期间简单地扩展RVC指令，简化验证并最小化对现有微架构的修改。

* 编译器可以不知道RVC扩展，将代码压缩留给汇编器和链接器，尽管压缩感知编译器通常能够产生更好的结果。

### 寄存器映射

压缩指令中经常使用的8个寄存器映射到寄存器`x8`到`x15`：

| RVC寄存器号 | 整数寄存器 | ABI名称 | 浮点寄存器 | 浮点ABI名称 |
|-------------|------------|---------|------------|-------------|
| 000 | x8 | s0 | f8 | fs0 |
| 001 | x9 | s1 | f9 | fs1 |
| 010 | x10 | a0 | f10 | fa0 |
| 011 | x11 | a1 | f11 | fa1 |
| 012 | x12 | a2 | f12 | fa2 |
| 101 | x13 | a3 | f13 | fa3 |
| 110 | x14 | a4 | f14 | fa4 |
| 111 | x15 | a5 | f15 | fa5 |

> **注释：**
> RISC-V ABI被更改以使频繁使用的寄存器映射到寄存器x8-x15。这通过具有连续的自然对齐的寄存器号集合简化了解压缩解码器，并且也与只有16个整数寄存器的RV32E和RV64E基础ISA兼容。

## 压缩指令格式

压缩指令使用九种格式。CR、CI和CSS可以使用32个RVI寄存器中的任何一个，但CIW、CL、CS、CA和CB限制为其中的8个。

### 格式详细说明

**CR格式（寄存器）：**
- 用于寄存器到寄存器的操作
- 可以访问所有32个寄存器
- 示例：C.ADD, C.MV, C.JR

**CI格式（立即数）：**
- 用于立即数操作
- 6位立即数，符号扩展
- 示例：C.ADDI, C.LI, C.LWSP

**CSS格式（栈相对存储）：**
- 专用于栈相对存储
- 使用栈指针作为基址
- 示例：C.SWSP, C.SDSP

**CIW格式（宽立即数）：**
- 用于8位立即数
- 主要用于C.ADDI4SPN
- 允许较大的栈偏移

**CL/CS格式（加载/存储）：**
- 用于基于寄存器的加载/存储
- 限制为8个基址和数据寄存器
- 立即数根据数据大小缩放

**CA格式（算术）：**
- 用于双操作数算术操作
- 限制为8个寄存器
- 示例：C.SUB, C.AND, C.OR

**CB格式（分支/算术）：**
- 用于条件分支和一些算术操作
- 示例：C.BEQZ, C.BNEZ, C.SRLI

**CJ格式（跳转）：**
- 用于无条件跳转
- 11位偏移量
- 示例：C.J, C.JAL

## 加载存储指令

为了增加16位指令的覆盖范围，数据传输指令使用按数据字节大小缩放的零扩展立即数：字×4，双字×8，四字×16。

RVC提供两种加载和存储变体。一种使用ABI栈指针x2作为基址并可以针对任何数据寄存器。另一种可以引用8个基址寄存器之一和8个数据寄存器之一。

### 基于栈指针的加载和存储

**C.LWSP（从栈加载字）：**
```
C.LWSP rd, offset
等价于: LW rd, offset(x2)

立即数编码: offset[5:2|7:6]，以4字节为单位
范围: 0-252 (4的倍数)

示例: C.LWSP x1, 8  # x1 = M[x2 + 8]
```

**C.SWSP（存储字到栈）：**
```
C.SWSP rs2, offset  
等价于: SW rs2, offset(x2)

立即数编码: offset[5:2|7:6]，以4字节为单位
范围: 0-252 (4的倍数)

示例: C.SWSP x5, 12  # M[x2 + 12] = x5
```

### 基于寄存器的加载和存储

**C.LW（加载字）：**
```
C.LW rd', offset(rs1')
等价于: LW rd', offset(rs1')

立即数编码: offset[5:3|2|6]，以4字节为单位
范围: 0-124 (4的倍数)

示例: C.LW x8, 4(x9)  # x8 = M[x9 + 4]
```

**C.SW（存储字）：**
```
C.SW rs2', offset(rs1')
等价于: SW rs2', offset(rs1')

立即数编码: offset[5:3|2|6]，以4字节为单位  
范围: 0-124 (4的倍数)

示例: C.SW x10, 8(x11)  # M[x11 + 8] = x10
```

## 控制转移指令

### 无条件跳转

**C.J（无条件跳转）：**
```
C.J offset
等价于: JAL x0, offset

立即数编码: 11位有符号偏移量，以2字节为单位
范围: ±2 KiB

示例: C.J label  # PC = PC + offset_to_label
```

**C.JAL（跳转和链接，仅RV32）：**
```
C.JAL offset  
等价于: JAL x1, offset

在RV64中，此编码用于C.ADDIW

示例: C.JAL function  # x1 = PC + 2, PC = PC + offset
```

### 间接跳转

**C.JR（间接跳转）：**
```
C.JR rs1
等价于: JALR x0, 0(rs1)

示例: C.JR x1  # PC = x1 (返回)
```

**C.JALR（间接跳转和链接）：**
```
C.JALR rs1
等价于: JALR x1, 0(rs1)

示例: C.JALR x5  # x1 = PC + 2, PC = x5
```

### 条件分支

**C.BEQZ（分支如果等于零）：**
```
C.BEQZ rs1', offset
等价于: BEQ rs1', x0, offset

立即数编码: 8位有符号偏移量，以2字节为单位
范围: ±256字节

示例: C.BEQZ x8, label  # if (x8 == 0) PC = PC + offset
```

**C.BNEZ（分支如果不等于零）：**
```
C.BNEZ rs1', offset
等价于: BNE rs1', x0, offset

示例: C.BNEZ x9, label  # if (x9 != 0) PC = PC + offset
```

## 整数计算指令

### 立即数算术

**C.ADDI（立即数加法）：**
```
C.ADDI rd, imm
等价于: ADDI rd, rd, imm

立即数: 6位有符号立即数
范围: -32到+31

示例: C.ADDI x1, 5  # x1 = x1 + 5
```

**C.ADDI4SPN（栈指针加法）：**
```
C.ADDI4SPN rd', imm
等价于: ADDI rd', x2, imm

立即数: 10位无符号立即数，以4字节为单位
范围: 4-1020 (4的倍数)

示例: C.ADDI4SPN x8, 16  # x8 = x2 + 16
```

**C.LI（加载立即数）：**
```
C.LI rd, imm
等价于: ADDI rd, x0, imm

示例: C.LI x1, 10  # x1 = 10
```

**C.LUI（加载高位立即数）：**
```
C.LUI rd, imm
等价于: LUI rd, imm

立即数: 6位有符号立即数 → imm[17:12]

示例: C.LUI x1, 1  # x1 = 0x1000
```

### 寄存器算术

**C.ADD（寄存器加法）：**
```
C.ADD rd, rs2
等价于: ADD rd, rd, rs2

示例: C.ADD x1, x2  # x1 = x1 + x2
```

**C.SUB（寄存器减法）：**
```
C.SUB rd', rs2'
等价于: SUB rd', rd', rs2'

示例: C.SUB x8, x9  # x8 = x8 - x9
```

**C.AND, C.OR, C.XOR（逻辑操作）：**
```
C.AND rd', rs2'   # rd' = rd' & rs2'
C.OR rd', rs2'    # rd' = rd' | rs2'  
C.XOR rd', rs2'   # rd' = rd' ^ rs2'
```

### 移位指令

**C.SLLI（逻辑左移立即数）：**
```
C.SLLI rd, imm
等价于: SLLI rd, rd, imm

移位量: 5位（RV32）或6位（RV64）

示例: C.SLLI x1, 2  # x1 = x1 << 2
```

**C.SRLI（逻辑右移立即数）：**
```
C.SRLI rd', imm
等价于: SRLI rd', rd', imm

示例: C.SRLI x8, 3  # x8 = x8 >> 3 (无符号)
```

**C.SRAI（算术右移立即数）：**
```
C.SRAI rd', imm
等价于: SRAI rd', rd', imm

示例: C.SRAI x9, 4  # x9 = x9 >> 4 (有符号)
```

### 其他指令

**C.MV（移动寄存器）：**
```
C.MV rd, rs2
等价于: ADD rd, x0, rs2

示例: C.MV x1, x2  # x1 = x2
```

**C.NOP（空操作）：**
```
C.NOP
等价于: ADDI x0, x0, 0

示例: C.NOP  # 什么都不做
```

## 断点指令

**C.EBREAK（断点）：**
```
C.EBREAK
等价于: EBREAK

用于调试器断点和半主机环境

示例: C.EBREAK  # 触发调试器
```

## 实现考虑

### 解码简化

压缩指令设计为在解码阶段简单扩展为32位指令：

1. **寄存器映射**：3位寄存器字段直接映射到x8-x15
2. **立即数重组**：立即数字段可以通过简单的布线重组
3. **操作码映射**：每个压缩指令对应一个明确的32位指令

### 代码密度收益

RVC通过以下方式实现代码大小减少：

- **常见模式**：压缩最频繁的指令模式
- **栈操作**：专门优化栈相关操作  
- **小立即数**：大多数立即数值都很小
- **寄存器局部性**：程序倾向于使用少数寄存器

### 性能影响

压缩指令的性能优势：

- **指令带宽**：减少25-30%的指令获取带宽
- **缓存效率**：指令缓存未命中减少20-25%
- **能耗降低**：较少的内存访问和更好的缓存利用率

---

C扩展通过提供常见操作的紧凑编码显著改善了RISC-V的代码密度，同时保持了与基础ISA的完全兼容性。其设计平衡了压缩效率、实现复杂性和解码速度，使其成为从嵌入式系统到高性能处理器的理想选择。
